一. 项目开发说明

    软件项目:完整的具有一定功能的代码集合

    软件项目的一般流程: 需求分析-->概要设计-->项目规划-->详细设计 --> 编码测试 --> 项目测试 --> 调试完善 --> 项目发布 --> 后期维护

    需求分析: 确定用户的真实需求
    1. 确定需求,项目的基本功能
    2. 确定项目难度及可行性
    3. 完成需求文档进行确认

    概要设计 : 对项目进行初步分析和整体设计
    1. 确定项目架构
    2. 进行整体模块划分,确定基本技术和使用框架
    3. 形成概要设计文档,指导开发工作

    项目计划 : 对项目开发时间和流程做规划
    1. 确定开发工作的先后顺序
    2. 确定时间轴和重要事件的里程碑
    3. 人员分工
    4. 形成甘特图和思维导图

    详细设计: 具体的项目实现手册
    1. 形成详细设计文档 : 思路,逻辑流程,数据机构,技术点说明,代码说明...

    编码测试: 按照预定计划实现代码
    1. 代码编写
    2. 技术攻关
    3. 代码测试

    项目测试:对项目整体进行测试
    1. 安全性,工能完善,跨平台..
    2. 完成测试报告

    调试修改:
    1.根据测试结果进行修改完善

    项目发布:
    1. 交付用户,或者发布
    2. 编写说明文档

    注意事项:

    * 项目计划的重要性
    * 按时完成项目是项目经理最重要的任务
    * 项目过程中的冲突


    项目工具: 

    文档编写 : word   markdown 
    项目流程图: Mindmanager  visio
    项目管理: project
    代码管理: svn  git


二. 网络聊天室

功能: 类似qq群聊

1. 进入聊天室需要输入姓名,姓名不能重复
2. 进入聊天室会向其他人发送通知
    xxx 进入聊天室
3. 一个人发消息群里其他人均可收到
    xxx说:xxxxxxxxxxxxx
4. 某人退出聊天室其他人也会收到通知
    xxx 退出聊天室
5. 管理员可以发送管理员消息,此时所有人都可以收到
    管理员消息 : xxxxxxxx

确定技术模型

  1. 发送消息: 转发,先发给服务器再转发其他人
  2. 套接字 : UDP套接字
  3. 收发关系处理: 多进程,让消息收发互不干扰

* 网络连接

* 实现具体功能

  进入聊天室
  聊天
  退出聊天室
  管理员消息

* 整体测试


ftp 文件服务器

功能 : 1. 服务端客户端两部分,要求启动服务器后可以
          有多个客户端同时登陆
       2. 客户端可以查看服务器文件库中的文件(只查看到文件库中普通文件即可)

       3. 客户端可以从文件库下载文件到本地
       4. 客户端可以上传本地文件到文件库
       5. 使用print打印一定格式作为命令输入提示选项

技术分析 :  1. 并发  多进程  多线程
            2. tcp套接字
	    3. 类封装功能

工作步骤:  先搭建网络连接
           设计类
	   将功能写在类中,逐一实现并测试

具体功能:  1. 搭建网络
   
           2. 查看文件列表

	   3. 上传文件

	   4. 下载文件
